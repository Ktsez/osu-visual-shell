# 项目方向

## 项目定位

`osu! Visual Shell` 是一个本地运行的音乐视觉外壳。

它的目标是提供一个接近 osu! 主菜单音乐舞台的视觉体验：

- 中央圆盘
- 歌曲封面
- 背景图
- 能量柱
- 侧灯
- 冲击波
- 星星喷泉
- 本地音乐播放控制

项目读取用户本机的普通音乐文件夹或 osu! Songs 文件夹，在浏览器中播放音乐并生成视觉效果。

## 项目边界

这个项目不是 osu! 客户端。

明确不做：

- 不做游戏模式
- 不做打图
- 不做计分
- 不做排行榜
- 不做账号系统
- 不做联网谱面下载
- 不内置音乐
- 不内置 osu! 官方资源
- 不接第三方音乐平台，除非用户之后明确要求

## 隐私边界

默认只在本机运行：

```text
127.0.0.1:4173
```

项目只读取用户主动选择或填写的本地文件夹。

项目不上传：

- 音乐
- 谱面
- 背景图
- metadata
- 本地路径列表

只有用户明确设置：

```bash
HOST=0.0.0.0 npm start
```

才允许局域网访问。

## 技术方向

保持当前技术栈：

- 原生 HTML
- 原生 CSS
- 原生 JavaScript
- Canvas
- Web Audio API
- Node.js 本地服务

短期不要引入大型框架。

## 维护方向

优先级：

1. 稳定现有体验
2. 清理模块结构
3. 增加测试覆盖
4. 提升性能
5. 增加可控设置
6. 最后再增加新功能

## 不推荐方向

不要为了代码结构好看而一次性重写视觉系统。

不要为了“像 osu!”而直接搬运 osu! 官方资源。

不要把本地文件服务默认暴露到局域网。

不要在未验证性能的情况下增加更多粒子、模糊、阴影或大面积滤镜。

不要把本项目扩展成完整音乐播放器平台。

## 推荐路线

下一阶段建议继续做 beta 稳定化：

- 逐步拆小 `src/visual/renderer.js`
- 增加 parser 和纯函数测试
- 加入真实截图
- 加入性能监控面板
- 记录默认参数和视觉 preset
- 再考虑桌面打包
